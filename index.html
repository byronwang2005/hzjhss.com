环境安装指南 Node.js · npm · Homebrew

Node.js 环境安装教程

按系统完成 Node.js 与 npm 的安装，并在终端验证命令可用。macOS 用户建议先安装 Homebrew，再用它管理 Node.js。

推荐路径 macOS: Homebrew → Node.js Windows: 官方 LTS 安装包 Linux: apt 或 nvm
00

安装前确认

准备一个终端，并优先选择 LTS 长期支持版。LTS 通常更稳定，适合日常开发与教学环境。

系统

macOS、Windows、Ubuntu / Debian 或其他常见 Linux 发行版。

终端

macOS 使用“终端”，Windows 使用 PowerShell，Linux 使用系统 Shell。

目标

安装后能正常执行 node -v 与 npm -v。

01

Homebrew 安装教程

Homebrew 是 macOS 常用的软件包管理器。安装后可以用一条命令安装、升级和卸载 Node.js。

  1. 打开 macOS 的“终端”。
  2. 执行 Homebrew 官方安装脚本，按提示输入电脑密码。
  3. 安装结束后，根据终端提示把 Homebrew 加入 shell 环境。
  4. 执行验证命令，确认 brew 可用。
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew -v

如果提示 brew: command not found，复制安装完成时终端给出的 eval "$(/opt/homebrew/bin/brew shellenv)" 相关命令后再验证。

02

macOS 安装 Node.js

Homebrew 安装完成后，用它安装 Node.js。npm 会随 Node.js 一起安装。

  1. 在终端执行安装命令。
  2. 等待安装完成。
  3. 检查 Node.js 与 npm 版本号。
brew install node
node -v
npm -v
03

Windows 安装 Node.js

访问 Node.js 官网 下载 LTS 版本的 Windows Installer。安装过程保持默认选项即可。

  1. 下载 LTS 版本安装包。
  2. 运行安装程序，并保留“添加到 PATH”的默认设置。
  3. 安装结束后重新打开 PowerShell。
  4. 执行版本检查。
node -v
npm -v
04

Linux 安装 Node.js

Ubuntu / Debian 可以先使用系统包管理器安装。多项目开发时，再考虑用 nvm 管理多个版本。

  1. 打开终端。
  2. 更新软件源并安装 Node.js 与 npm。
  3. 确认版本号输出正常。
sudo apt update
sudo apt install nodejs npm
node -v
npm -v
05

验证运行环境

最后运行一段最小 JavaScript，确认 Node.js 可以执行脚本。

node -e "console.log('Node.js is ready')"

终端输出 Node.js is ready，说明环境已经可用。

Node.js 环境安装教程 更新后请重新打开终端再验证命令。
